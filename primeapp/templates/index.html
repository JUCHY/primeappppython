The Process


Uses the Miller Rabin Probability test and hence trades accuracy for computation speed
Contains 3 steps:

  • Step 1: Find n-1 = 2k * m, where n chosen number, and k and m are random integers. We start by dividing the number by 21, then 22, onward, until we find a number 2k that n-1 is not divisible by. Then we take the k, and m, from the previous number and send that to step 2.
  • Step 2: We find an a where 1 < a < n-1. For this program, we used to as our a in all cases. With that we calculate 2mmod n, and that is sent to the last step
  • Step 3: The program in its first iteration checks if 2mmod n is equal to 1 or -1, if it is not, 2m is squared then the program checks if (2m)2>mod n is equal to -1. If equal to -1, the number is a prime, if equal to 1, it is not prime. if it is neither, the recursion goes on until it is either 1 or -1, or it hits its 100th iteration, therein the program decides that it is composite
